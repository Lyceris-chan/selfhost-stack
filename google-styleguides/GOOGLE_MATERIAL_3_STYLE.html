Google Material Design 3 Style Guide

1 Introduction

Material Design 3 (also known as Material You) is the latest version of Googleâ€™s open-source design system. It is designed to be personal, adaptive, and expressive. This document summarizes the core UI design rules and principles for Material Design 3.

2 Foundations

2.1 Accessibility

  • Contrast: Minimum contrast ratio of 4.5:1 for normal text and 3:1 for large text.
  • Touch Targets: Minimum size of 48x48dp, with at least 8dp of space between targets.

2.2 Design Tokens

M3 uses design tokens to store style values. Tokens are categorized into:

  • Reference tokens: Absolute values (e.g., Palette.Blue.40).
  • System tokens: Context-specific roles (e.g., Sys.Color.Primary).
  • Component tokens: Component-specific attributes (e.g., Comp.Button.Container.Color).

3 Color

3.1 Principles

  • Dynamic Color: M3 uses dynamic color algorithms to create accessible and harmonious color schemes based on a user's wallpaper or a specific seed color.
  • Tone-based: Color is defined by its tonal value (luminance) rather than just hue and saturation, ensuring consistent contrast across different themes.

3.2 Color Roles

Each color in an M3 theme has a specific role:

RoleDescription
PrimaryUsed for key components like FABs, prominent buttons, and active states.
SecondaryUsed for less prominent components like filter chips and selection controls.
TertiaryUsed for accenting and contrasting with primary/secondary colors.
ErrorUsed to communicate error states.
SurfaceUsed for backgrounds of cards, sheets, and dialogs.

5 Layout

5.1 8dp Grid

All components and layout elements should align to an 8dp square grid. For smaller elements like icons and type, a 4dp grid can be used.

5.2 Spacing

Spacing between elements should be in increments of 4dp or 8dp (e.g., 8dp, 16dp, 24dp, 32dp).

6 Components

6.1 Action

  • Buttons: Corner radius is typically 20dp (fully rounded for small buttons). High-emphasis buttons use the Primary color.
  • FAB: Corner radius is 12dp for small/standard FABs and 28dp for large FABs.
  • Chips: Used for input, selection, filtering, or triggering actions. Types include assist, filter, input, and suggestion.

6.2 Containment

  • Cards: Corner radius is 12dp. Left/right padding is 16dp. Types include Elevated, Filled, and Outlined.
  • Dialogs: Corner radius is 28dp. Padding is 24dp.
  • Bottom Sheets: Anchored to the bottom of the screen; used for secondary or modal content.

6.3 Communication

  • Badges: Display notifications, counts, or status on icons. Content is limited to 4 characters.

6.4 Navigation

  • Navigation Bar: For primary destinations on compact/medium screens.
  • Navigation Drawer: For larger screens (expanded/large/extra-large); being replaced by expanded navigation rails.
  • Tabs: Organize content across different screens or views; primary for main destinations, secondary for related content.

6.5 Information

  • Lists: Help users find and act on items; should be short, scannable, and logically ordered.
  • Menus: Present a temporary set of actions relative to a component or field.

6 Writing

6.1 Voice and Tone

  • Direct: Use clear and concise language.
  • Helpful: Provide useful information to guide the user.
  • Human: Use a friendly and professional tone.

6.2 Capitalization

Use Sentence case for most UI text, including titles, labels, and buttons. Title Case should be avoided unless it's a brand name.
